# Kzyo 每日简报独立子站设计

## 1．背景与目标

参考 `https://digest.zhangtenggan.cn/` 的产品形态（每日自动聚合 + AI 摘要 + 列表式简报），为 Kzyo 建一个每日简报子站，发布于 `https://alainouyang.github.io/digest`。

- 只复刻形态，不复制任何第三方站点的内容、摘要文字或策展成果。
- 简报本身作为一个持续运行的 AI 应用作品，支撑 Kzyo「AI 应用与效率工具」的副定位。
- 主站保持编辑型低频气质，简报的每日自动提交与部署完全隔离在独立仓库。

## 2．仓库与部署架构

- 新建独立仓库 `AlainOUYANG/digest`，GitHub Pages 项目页发布到 `alainouyang.github.io/digest`。
- 仓库包含轻量 Astro 静态站与每日流水线脚本两部分。
- GitHub Actions 定时任务每天北京时间 19:00（UTC 11:00）运行：抓取 → 筛选摘要 → 提交当期 markdown → 构建部署。
- 主站仓库唯一的改动是在页脚或 AI 实验室加一个指向 `/digest` 的入口链接，单独小提交。
- 本地开发目录为主站仓库的同级目录 `~/digest`。

## 3．信息源与配置

三组源，全部存放于仓库内可编辑的配置文件：

1. **AI 技术博客**：提取 `vigorX777/ai-daily-digest` 内嵌的约 90 个 RSS 源 URL（源列表属事实数据；该仓库无 License，流水线代码全部自研，不复制其实现）。
2. **因果推断与营销科学**：arXiv RSS（`stat.ME`、`econ.EM` 等分类）加精选博客初始清单，实施时拟定，配置文件可随时增删。
3. **即刻简报**：配置文件中的即刻用户 ID 列表 → RSSHub `/jike/user/:id` 路由抓取动态。

即刻名单引导（一次性，非运行时依赖）：张腾甘的即刻关注列表（52 人）需登录才能查看，无公开接口；等价替代方案是扫描 `digest.zhangtenggan.cn` 公开归档中即刻栏目出现过的用户（即其关注且活跃的子集），从条目内的原帖链接解析作者 ID，去重后写入配置文件。若归档中无法解析出足够的原帖链接，退路是用户在即刻 App 内查看该关注列表并手动摘录名单，再映射为 ID。名单落成静态配置后由用户自行维护。

## 4．每日流水线

自研 Node 脚本，流程：

1. 三组源并行抓取；即刻部分在 Action 内用 `npx rsshub` 自托管实例抓取，公共 RSSHub 实例作退路。
2. 近 24 小时时间窗过滤。
3. 调用火山引擎方舟（OpenAI 兼容接口，`base_url` + 模型名可配，key 存 GitHub Actions secrets）对候选条目按相关性与质量评分筛选。
4. 为入选条目生成中文摘要；每条必须附原文链接与作者署名，不全文转载。
5. 输出当期 markdown（三栏目结构），提交到内容目录，触发构建部署。

## 5．站点呈现

- 首页展示最新一期；归档页按日期倒序列出全部期数；每期页面为三栏目列表式排版。
- 输出 RSS。
- 视觉沿用主站的暖白编辑型系统（复制 tokens），中文文楷、西文 Source Sans 3。
- 站点身份为 `Kzyo`，遵守主站既有匿名边界：不出现真名变体、个人照片、雇主与公司业务信息。

## 6．错误处理与幂等

- 单个源抓取失败：跳过该源，当期页脚标注「N 个源本期不可用」。
- LLM 调用失败：重试一次；仍失败则该栏目降级为「仅标题 + 链接、无摘要」。
- 整个 Action 失败：当天缺期，不影响已发布内容。
- 同日重跑覆盖当期，流水线幂等。

## 7．验收标准

- 流水线单测覆盖时间窗过滤、RSS 解析、markdown 生成。
- 构建契约：缺必填字段（标题、日期、来源链接）构建失败。
- 上线流程：先手动触发跑通首期 → Playwright 冒烟（核心路由、移动端、控制台、横向溢出）→ 用户目检首期内容质量 → 才开启每日定时。
- 版权卫生：抽查条目确认均为摘要 + 署名 + 原文链接，无全文转载。
